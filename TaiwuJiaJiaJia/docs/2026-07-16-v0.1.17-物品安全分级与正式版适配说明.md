# 太吾加加加 v0.1.17 物品安全分级与正式版适配说明

> 文档日期：2026-07-16
> 目标版本：v0.1.17
> 状态：已实现，待游戏内回归验证

## 一、改版背景

当前版本使用官方 `ItemConfigHelper.AllConfigs` 读取 13 类、共 4500 个基础物品模板，并非只包含抢先体验版物品。

但当前安全规则将所有 `IsSpecial=true` 的物品统一隐藏并禁止添加，导致大量正式版内容被误归类。

当前数量对应如下：

- 官方基础物品模板：4500
- `IsSpecial=true` 的物品：863
- 其他任务锁或名称规则拦截：9
- 当前默认展示：3628

正式版新增的玄品宝物、门派后传奖励等常被官方标记为 `IsSpecial=true`，但它们不全是会直接推进任务的关键道具。新版不再仅依赖 `IsSpecial` 做一刀切判断。

## 二、改版目标

1. 正式版的所有物品模板都能被搜索和查看。
2. 剧情锁定、任务触发、虚拟占位等高风险物品始终禁止添加。
3. 玄品宝物及不带任务锁的特殊奖励允许在明确告知风险后添加。
4. 不因游戏更新继续使用旧版物品缓存。
5. 所有高风险操作都可识别、可阻止、有警告日志，不记录普通浏览日志。

## 三、新的三级安全分类

### A 级：常规物品

符合以下条件时，允许直接添加：

- `TaskLock` 为空。
- 不在剧情物品硬锁名单中。
- 不会触发通用事件、任务完成或剧情参数变更。
- 不是虚拟物品或系统内部占位模板。
- `IsSpecial=false`，或经明确白名单判定为普通可获得物品。

行为：

- 使用当前数量输入。
- 数量限制为 `1 - 9999`。
- 点击“添加”后直接调用官方创建物品接口。

### B 级：风险物品

符合以下条件且未命中 A/C 级规则时，列为风险物品：

- `IsSpecial=true`，但 `TaskLock` 为空。
- 正式版玄品宝物，即 `MysteryEffectId >= 0`。
- 门派剧情或后传的唯一奖励，但物品本身不携带任务锁。
- `Transferable=false`、`AllowRandomCreate=false` 等表明官方不希望它在常规掉落或交换流程中出现的物品。

行为：

- 列表和搜索中正常显示。
- 卡片显示黄色警告图标和“需确认”标记。
- 添加数量强制为 `1`，不允许批量添加。
- 按钮文案改为“风险添加”。
- 点击后必须经过独立确认弹窗。

确认弹窗必须说明：

- 该物品可能是唯一剧情奖励或玄品宝物。
- 提前获得可能破坏正常解锁节奏。
- 重复获得可能影响平衡、仓库数据或后续奖励判定。
- Mod 只创建物品，不会自动回滚或修复由此引起的流程变化。

确认按钮使用明确文案：

- 取消
- 仍然添加 1 个

### C 级：剧情锁定物品

命中任一条件时，始终禁止添加：

- `TaskLock` 不为空。
- 官方接口明确判定为门派主线、任务信物或关键道具。
- 物品会直接触发通用事件或作为剧情参数判定条件。
- 虚拟物品、系统占位物品、临时事件容器。
- 硬锁名单中明确标记为不可创建的物品。

行为：

- 物品仍可搜索、选中并查看详情。
- 卡片显示红色锁定图标和“剧情锁定”标记。
- 添加按钮始终禁用，不提供“我已知晓”类绕过入口。
- 详情面板显示命中的具体锁定原因。

## 四、初始分类顺序

分类器应按以下顺序执行，不可调换：

1. 验证模板有效性。
2. 检查虚拟物品、事件触发和系统占位标记。
3. 检查 `TaskLock`。
4. 检查官方剧情物品 Helper 和项目硬锁名单。
5. 检查 `MysteryEffectId`。
6. 检查 `IsSpecial`、`Transferable`、`AllowRandomCreate` 等风险信号。
7. 剩余物品归入 A 级。

优先级固定为：`C 级 > B 级 > A 级`。例如一件玄品同时带有 `TaskLock`，必须归入 C 级，不得因为它是玄品而放宽。

## 五、界面调整

### 5.1 筛选项

移除含义不清的“显示任务/特殊物品”复选框，改为安全状态分段筛选：

- 全部
- 可添加
- 需确认
- 剧情锁定

搜索在全量物品模板上执行。输入关键词后会暂时忽略分类、品级和安全状态筛选，避免物品因当前筛选条件而表现为“不存在”；清空关键词后恢复原筛选条件。

输入框聚焦期间暂停呼出热键检测，避免玩家把热键设置为字母后，输入同一字母导致窗口关闭。界面同时提供“搜索”和“清空”按钮。

### 5.2 统计信息

列表顶部显示：

```text
物品配置：4500  |  可添加：X  |  需确认：Y  |  剧情锁定：Z
```

数字必须来自当前游戏实时配置，不在代码中写死。

### 5.3 卡片与详情

卡片和详情面板增加：

- 安全等级。
- 风险原因。
- 是否为玄品。
- 是否存在任务锁。
- 是否允许交换。
- 是否允许常规生成。
- 对剧情锁定物品显示精确的 `type/id`，方便排查，但不提供添加入口。

## 六、缓存升级

将磁盘缓存从 `ItemCache.v3` 升级为 `ItemCache.v4`。

新缓存头至少保存：

- Mod 缓存格式版本。
- `GameData.Shared.dll` 的 Module Version ID（MVID）。
- 扫描到的物品总数。
- 安全分类规则版本。

自动失效条件：

- MVID 变化。
- 缓存格式版本变化。
- 安全分类规则版本变化。
- 缓存数据不完整或解析失败。

失效后应在首次打开界面时自动重建，不再要求玩家手动点击“刷新”。手动刷新按钮仍保留，用于故障排查。

## 七、日志

仅记录 warning/error：

- 缓存因游戏版本变化而自动失效：warning。
- 缓存解析失败：warning。
- 尝试添加 C 级剧情锁定物品：warning。
- 用户确认添加 B 级风险物品：warning，记录物品名、`type/id` 和风险类型。
- 官方创建物品接口返回异常：error。

普通物品添加、搜索、筛选、悬停等操作不记日志。

## 八、兼容性与存档影响

- 本次不修改已存在的物品实例。
- 本次不修改玩家已获得的任务奖励。
- 缓存升级只影响 Mod 物品列表，不写入游戏存档。
- 更新 Mod 后，旧 `ItemCache.v3` 将被忽略并自动生成 v4 缓存。
- 玩家主动确认添加的 B 级物品会成为正常存档物品，卸载 Mod 不会自动删除它。

## 十、本版暂不处理

- 不创建带自定义精制效果的物品实例。
- 不复制玩家已持有物品的耐久、毒素、精制、改造等实例数据。
- 不解锁与物品对应的剧情、成就、百晓册或任务进度。
- 不对玩家已经添加的风险物品提供自动删除或回滚。

## 十一、实现顺序

1. 将 `Blocked/BlockReason` 替换为三级安全模型。
2. 实现 C 级硬锁判定和不可绕过的后端保护。
3. 实现 B 级风险确认及单件数量限制。
4. 改造筛选、卡片、详情和统计显示。
5. 升级并自动失效磁盘缓存。
6. 补充 warning/error 日志。
7. 构建、打包并在新档、中途加载存档与游戏热更时分别验证。
