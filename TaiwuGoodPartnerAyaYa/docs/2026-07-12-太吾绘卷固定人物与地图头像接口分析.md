# 太吾绘卷固定人物与地图头像接口分析

> 记录日期：2026-07-12  
> 目标 Mod：太吾好伙伴阿雅  
> 目的：梳理“固定人物创建、地块角色列表、大地图头像/任务标记、点击交互、图片资源字段”的接口边界，避免后续把不同 UI 入口混为一谈。

## 结论先行

目前能确认太吾里至少有两类容易混淆的角色入口：

1. **地块人物列表里的特殊人物**
   - 后端来源是 `MapBlockData.FixedCharacterSet`。
   - 前端请求当前地块人物时，后端把它组装到 `MapBlockCharacterList.SpecialCharacters`。
   - 玩家到达或选中该地块后，可以在地块人物列表里点击交互。

2. **大地图上单独显示的头像/特殊角色标记**
   - 不等同于 `MapBlockCharacterList.SpecialCharacters`。
   - 后端提供了 `MapDomain.QueryFixedCharacterLocation(templateId)` 这类定位接口。
   - 原版任务系统会通过 `TaskInfoItem.CharacterTemplateId` 查找固定人物位置，并把位置写进 `TaskDisplayData.TargetLocation`。
   - 因此前端大地图头像/目标标记很可能是“固定人物位置 + 任务/剧情目标显示”组合出来的，而不是只要进入 `FixedCharacterSet` 就必然显示。

阿雅当前实现更接近第 1 类：作为固定人物进入当前地块的特殊人物列表，并通过点击触发专属交互。要做成第 2 类，还需要继续接入轻量任务或剧情目标显示链路。

## 固定人物模板字段

反编译位置：


与固定人物和头像相关的字段包括：

| 字段 | 用途判断 |
| --- | --- |
| `CreatingType` | `0` 表示固定人物。点击时会走 `FixedCharacterClicked`。 |
| `FixedAvatarName` | 固定人物静态头像资源名，可能参与地图/事件头像绘制。 |
| `FixedAvatarSpineName` | 固定人物 Spine 动态立绘资源名。 |
| `FixedAvatarSpineSkin` | Spine 皮肤名。 |
| `FixedCharacterShowNameOnMap` | 固定人物是否在地图相关 UI 上显示名字。 |
| `AvatarDataPath` | 人物通用头像数据路径，可能影响人物详情/列表头像。 |
| `CanOpenCharacterMenu` | 是否允许打开完整人物菜单。特殊伙伴建议关闭。 |
| `CanSpeak` | 是否允许说话/交互。 |

阿雅当前模板：

```lua
CreatingType = 0
CanOpenCharacterMenu = false
CanSpeak = true
FixedCharacterShowNameOnMap = true
```

注意：

- `FixedCharacterShowNameOnMap = true` 更像是“允许地图相关 UI 显示名字”，不等于强制生成大地图头像标记。
- 要让自定义立绘真正被前端绘制，还需要确认 `FixedAvatarName`、`FixedAvatarSpineName`、`AvatarDataPath` 对 Mod 资源路径的支持方式。当前阿雅包内 PNG 还没有接入原版头像资源索引。

关键接口：

固定人物缓存逻辑：

```csharp
if (Config.Character.Instance[character2.GetTemplateId()].CreatingType == 0)
{
    _fixedCharactersCache.Add(character2.GetTemplateId(), value);

```

结论：

- 固定人物的稳定识别核心是 `Character.TemplateId`，不是运行时 `charId`。
- `12001` 是阿雅的固定人物模板 id，不是运行时角色 id。
- 与原版早期随机角色 id 不会直接冲突；主要风险是其他 Mod 也使用 `Character.TemplateId = 12001`。
- 对“太吾好伙伴”系列，建议保留一段自用模板 id 范围，例如 `12001` 起连续分配，并在文档中固定下来。一旦公开发布后不要轻易改模板 id，否则旧存档中的固定人物会与新模板脱节。

## 位置变化与地块特殊人物列表

反编译位置：

```text
/Users/shermer/Documents/Mod/decompiled/GameData/GameData.DomainEvents/Events.cs
/Users/shermer/Documents/Mod/decompiled/GameData/GameData.Domains.Map/MapDomain.cs
```

固定人物位置变化入口：

```csharp
public static void RaiseFixedCharacterLocationChanged(DataContext context, int charId, Location srcLocation, Location destLocation)
{
    DomainManager.Map.OnFixedCharacterLocationChanged(context, charId, srcLocation, destLocation);
    DomainManager.Character.OnFixedCharacterLocationChanged(context, charId, srcLocation, destLocation);
}
```

地图块数据更新：

```csharp
public void OnFixedCharacterLocationChanged(DataContext context, int charId, Location srcLocation, Location destLocation)
{
    if (srcLocation.IsValid())
    {
        MapBlockData block = GetBlock(srcLocation);
        if (block.RemoveFixedCharacter(charId))
        {
            SetBlockData(context, block);
        }
    }
    if (destLocation.IsValid())
    {
        MapBlockData block2 = GetBlock(destLocation);
        block2.AddFixedCharacter(charId);
        SetBlockData(context, block2);
        SetBlockAndViewRangeVisible(context, destLocation.AreaId, destLocation.BlockId);
    }
}
```

当前地块人物列表组装：

```csharp
public MapBlockCharacterList RequestMapBlockCharacterList(DataContext context, MapBlockData data)
{
    mapBlockCharacterList.SpecialCharacters =
        CharacterDisplayDataSorter(
            data?.FixedCharacterSet != null
                ? DomainManager.Character.GetCharacterDisplayDataList(data.FixedCharacterSet...)
                : ...
        );
}
```

结论：

- 让阿雅出现在地块特殊人物列表的正确做法是：
  1. 创建固定人物。
  2. 调用 `Events.RaiseFixedCharacterLocationChanged(context, charId, oldLocation, newLocation)`。
  3. 再调用 `character.SetLocation(newLocation, context)`。
- 这样会把阿雅加入目标地块的 `FixedCharacterSet`。
- 当前格角色列表请求时，阿雅会进入 `SpecialCharacters`。
- 这条链路确认的是“地块人物列表”，不是大地图单独头像标记。

点击角色后分流：

```csharp
public void OnCharacterClicked(DataContext context, int charId)
{
    Character element = DomainManager.Character.GetElement_Objects(charId);
    short templateId = element.GetTemplateId();
    if (element.GetCreatingType() == 0)
    {
        OnEvent_FixedCharacterClicked(charId, templateId);
    }
}
```

固定人物点击参数：

```csharp
public void OnEvent_FixedCharacterClicked(int arg0, short arg1)
{
    EventTrigger trigger = _eventManager.GetTrigger(4);
    trigger.ArgBox.Set(EventTriggerParameter.DefValue.CharacterId, arg0);
    trigger.ArgBox.Set(EventTriggerParameter.DefValue.CharacterTemplateId, arg1);
    trigger.OnEvent(trigger.ArgBox);
}
```

结论：

- 阿雅事件包里的专属交互应使用 `TriggerType = 4`。
- 条件检查要同时校验：
  - 点击的 `charId` 是当前记录的阿雅角色。
  - 点击的 `CharacterTemplateId` 是阿雅模板 id。
- 这样不会误响应其他固定人物。

反编译位置：

固定人物位置查询接口：

```csharp
[DomainMethod]
public Location QueryFixedCharacterLocation(short templateId)
{
    return DomainManager.Character.TryGetFixedCharacterByTemplateId(templateId, out character)
        ? character.GetLocation()
        : Location.Invalid;
}

[DomainMethod]
public Location QueryFixedCharacterLocationInArea(short templateId, short areaId)
{
    Location location = QueryFixedCharacterLocation(templateId);
    return (location.AreaId == areaId) ? location : Location.Invalid;
}
```

任务目标位置计算：

```csharp
foreach (short templateId in taskInfoItem.CharacterTemplateId)
{
    if (DomainManager.Character.TryGetFixedCharacterByTemplateId(templateId, out var character)
        && character.GetLocation().IsValid())
    {
        item.TargetLocation = character.GetLocation();
    }
}
```

结论：

- 大地图上“某个剧情/任务特殊人物在哪里”的标记，后端有现成定位接口。
- 原版任务系统会用 `TaskInfoItem.CharacterTemplateId` 把固定人物位置写入 `TaskDisplayData.TargetLocation`。
- 因此，如果阿雅要拥有“地图上单独的大头像/目标提示”，更可靠的路线可能是：
  1. 阿雅仍是固定人物模板 `12001`。
  2. 新增一个轻量任务或任务链，`CharacterTemplateId` 指向 `12001`。
  3. 任务目标显示由原版前端根据 `TargetLocation` 绘制。
  4. 玩家到达对应地块后，仍通过地块人物列表点击阿雅。
- 这部分还需要继续确认前端绘制规则，因为目前手头主要是后端和 Shared 配置反编译代码。

### 和阿雅需求的关系

门派剧情路线说明：大地图“特殊剧情目标显示”大概率不是单独的人物系统，而是任务/剧情目标显示系统的一部分。

阿雅如果要做成“地图上单独冒一个大头像/显眼标记”，比较稳的路线仍然是：

1. 阿雅先作为固定人物创建，模板 `Character.TemplateId = 12001`。
2. 新增一条轻量任务链，例如 `TaiwuGoodPartnerAyaLocationChain`。
3. 新增一条轻量任务，例如 `TaiwuGoodPartnerAyaLocationTask`。
4. `TaskInfo.CharacterTemplateId = { 12001 }`。
5. 后端在阿雅放置成功后调用：

```csharp
DomainManager.World.TriggerExtraTask(context, ayaTaskChainId, ayaTaskInfoId);
```

6. 阿雅加入、离去、卸载准备或失效时调用：

```csharp
DomainManager.World.FinishTriggeredExtraTask(context, ayaTaskChainId, ayaTaskInfoId);
```

### 风险和未确认点

- 原版门派升级任务配置本身没有使用非空 `TaskInfo.CharacterTemplateId`，多数任务位置可能来自 EventArgBox、固定门派地点或剧情特殊逻辑。
- `TaskInfo.CharacterTemplateId` 到 `TaskDisplayData.TargetLocation` 的后端链路是确认存在的，但前端最终画成“头像”、“普通任务标记”还是“任务目标光标”，还需要进游戏测试。
- 如果任务链使用 `SectMainStory` 分组，可能会混入门派剧情任务 UI；如果使用 `CustomizeTasks` 或 `OptionalTasks`，需要测试是否在当前阶段也显示地图目标。
- `TaskDisplayData` 有缓存。触发/完成任务会刷新任务数据，但如果只是阿雅移动位置，地图目标是否立即刷新需要测试。必要时可以在移动后重新触发同一任务或显式刷新任务状态，但这需要谨慎，避免任务列表闪动。
- 如果任务只是为了地图标记，它仍可能出现在任务列表里。因此任务标题、描述和图标要设计得很轻，不要像正式剧情任务一样打扰玩家。

### 暂定不接入的原因

这条路线目前值得做一个实验版，但不应该直接接进正式包。原因是现在已经确认“固定人物地块列表交互”可走官方路线，而“地图大头像”还差前端表现验证。下一步可以单独做 `0.1.3-task-marker-test`，只验证：

- 阿雅放置后是否出现地图任务目标。
- 标记显示类型是否接近门派剧情的大头像。
- 阿雅移动到太吾附近后，标记是否同步更新。
- 加入、离去、卸载准备后，任务是否能干净消失。

## 图片资源接口判断

已确认的资源相关字段：

```text
CharacterItem.FixedAvatarName
CharacterItem.FixedAvatarSpineName
CharacterItem.FixedAvatarSpineSkin
CharacterItem.AvatarDataPath
CharacterItem.FixedCharacterShowNameOnMap
```

这些字段说明原版固定人物存在至少两种头像/立绘资源路径：

- 静态固定头像：`FixedAvatarName`
- 动态 Spine 头像/立绘：`FixedAvatarSpineName` + `FixedAvatarSpineSkin`
- 通用人物头像数据：`AvatarDataPath`

当前还不能确认的点：

- Mod 自带 PNG 是否可以直接填入 `FixedAvatarName` 或 `AvatarDataPath`。
- 如果 `FixedAvatarName` 要求的是原版 AssetBundle 内部资源名，则需要前端资源包或原版资源索引接入。
- 地图大头像用的是 `FixedAvatarName`、`AvatarDataPath`，还是任务前端另有图标字段。
- 事件对白立绘能否直接引用 Mod 包内 `Assets/Images/*.png`，以及语言文件/事件包中对应的字段写法。

建议后续检查：

- 反编译前端 `Assembly-CSharp.dll`，搜索：
  - `FixedAvatarName`
  - `FixedAvatarSpineName`
  - `AvatarDataPath`
  - `QueryFixedCharacterLocation`

  - `SpecialCharacters`
- 如果前端代码里能看到具体资源加载函数，再决定阿雅 PNG 应该放在哪里、字段如何填写。

## 阿雅当前实现状态

当前包内实现更偏向“固定人物可交互”，不是完整任务头像追踪：

- 后端插件：
  - 读档后尝试创建阿雅固定人物。
  - 将阿雅移动到太吾当前地块。
  - 调用 `RaiseFixedCharacterLocationChanged` 刷新 `FixedCharacterSet`。
  - 记录 `CharacterId`、`Placed`、`SpawnDate`、`Joined`、`Dismissed`、`UninstallPrepared`。
  - 6 个月内未邀请同行则将阿雅移出地图并标记 `Dismissed`。

- 事件包：
  - `RecordEnterGame` 与 `NewGameMonth` 静默检查阿雅放置/离去状态。
  - `FixedCharacterClicked` 打开“与阿雅交谈”。
  - 玩家主动选择“随我同行”才调用入队。

当前不保证：

- 阿雅会在大地图上单独显示任务头像标记。
- 阿雅自定义 PNG 会被原版地图头像 UI 使用。
- 6 个月离去会产生月报提示。该功能暂缓，避免引入前端显示事件复杂度。
