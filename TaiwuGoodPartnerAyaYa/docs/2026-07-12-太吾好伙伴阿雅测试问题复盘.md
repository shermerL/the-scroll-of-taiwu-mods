# 太吾好伙伴阿雅测试问题复盘

更新时间：2026-07-12

## 当前测试结论

- 阿雅已经能作为固定人物被创建并放置到太吾附近地块，地图大头像和左侧人物列表都能出现。
- 玄灰/相枢净化流程已经能执行，但旧版结果文案里目标名显示为“此人”，原因是后端用 `Character.GetFullName()` 反射取字符串，而实际接口返回的不是字符串。已改为优先使用 `DomainManager.Character.GetName(charId, true)`。
- 加载时的 warning 来自事件包里的 `AyaPlacementTickEvent` 周期触发，它调用 `TaiwuGoodPartnerAya.TickPlacement` 时仍被 ModDomain 报 `Unable to call method ... with parameter and return value`。首版已移除这条周期事件链，改为只靠后端加载存档时放置阿雅。
- 截图中的阿雅立绘仍然来自源模板 `京畿女`，不是自定义 PNG。当前包里 PNG 还没有真正接入游戏前端资源索引。

## 固定人物与立绘

`Config.CharacterItem` 中与固定人物头像/立绘最相关的字段包括：

- `FixedAvatarName`
- `FixedAvatarSpineName`
- `FixedAvatarSpineSkin`
- `AvatarDataPath`

目前判断：这些字段大概率接收的是游戏资源索引名或资源路径名，而不是 Mod 文件夹里的裸 PNG 路径。要让自定义 PNG 真正显示在地图、事件对话和人物详情中，还需要进一步确认前端资源加载方式，或者接入前端插件/资源包。

阿雅至少需要三类图：

- 地图头像/左侧人物列表小头像。
- 事件对话右侧半身立绘。
- 人物详情页头像或半身展示图。

## 角色模板字段

本次模板已补充以下方向：

- `BaseAttraction = 900`：目标是让魅力显示到“天人”级别。具体阈值仍需游戏内验证，若仍不够可提高到 1000。
- `Health = 100`、`BaseMaxHealth = 100`：避免创建后显示重危。
- `FatalImmunity`、`DieImmunity`、`InnerInjuryImmunity`、`OuterInjuryImmunity`、`MindImmunity`、`FlawImmunity`、`AcupointImmunity`：用于降低死亡、伤病和负面状态对阿雅的影响。
- `Race = 0`、`PresetBodyType = 0`、`RandomAnimalAttack = false`、`XiangshuInfectedDemonBonus = false`：避免继承源模板或异常字段导致显示像非普通人物。

注意：模板字段主要影响新创建的固定人物。已在旧存档中创建过的阿雅，可能不会自动刷新全部外观和基础属性；测试模板变化时最好使用新档，或后续补一套运行时修正逻辑。

## 同道、战斗与异常交互

旧版开放了“随我同行”，调用的是 `DomainManager.Extra.GearMateJoinGroup`。测试后可以看到，阿雅进入队伍后会出现在普通人物详情页，并暴露原版同道/人物交互入口，例如“归灵”“铭刻”“交谈”等。

这些交互并不是阿雅 Mod 设计的一部分，而是原版人物系统在同道状态下自动显示的功能。由于阿雅是外部新增固定人物，目前还没有确认：

- 进入战斗是否会被当作普通同道参与战斗或触发助战指令。
- “归灵”“铭刻”等入口是否会修改阿雅或存档中的特殊字段。
- 遣散后是否会按原版同道逻辑回到太吾村，还是留下异常状态。

因此策略调整为：

- 阿雅保留为地图上的固定人物。
- 玩家通过点击阿雅进入专属事件，使用“净化当前地块目标”“随行”“暂候”和“卸载前准备”。
- 随行不再调用 `GearMateJoinGroup`，只记录本 Mod 的随行状态。
- 后端监听 `Events.OnTaiwuMove`，太吾移动后把阿雅同步到太吾当前地块；读档和月结后也会做兜底同步。

这样做的目标是保证玄灰净化功能能跟着太吾走，同时避开战斗、同道位、铭刻/归灵等系统带来的坏档风险。

## 遣散与卸载

“遣散回太吾村”暂不作为首版功能。原因是阿雅不是完整原版剧情人物，且进入同道后会暴露普通同道页面；直接离队可能留下不可预期状态。

当前只保留“准备卸载”：

- 仅在非战斗、非奇遇、非月结、非事件显示时允许执行。
- 执行后写入 `UninstallPrepared`，停止自动放置和净化。
- 若阿雅已经在同道中，后端会尝试 `GearMateLeaveGroup` 后再标记卸载准备。

若玩家已经在旧版中让阿雅入队，建议先在安全状态下使用“准备卸载”，再移除 Mod。后续若要正式支持“遣散”，应单独实现：离开队伍、移动到太吾村、关闭普通人物菜单入口、保留专属事件入口。

## 地图大头像机制

阿雅现在能显示地图大头像，主要依赖固定人物模板与：

- `CreatingType = 0`
- `FixedCharacterShowNameOnMap = true`
- 后端创建固定人物：`CreateFixedCharacter`
- 后端移动到太吾当前地块：`CharacterMove`

这说明门派剧情人物那类地图大头像，不一定必须先接任务链才能出现；固定人物本身也能进入地图头像展示。但任务/剧情目标显示可能仍有额外前端标记，后续如果要做“跟随任务指引”或“特殊头像状态”，还要继续研究任务链和门派剧情事件。

## 归灵是什么

从截图看，“归灵”出现在阿雅的人物详情交互面板里，不是本 Mod 写出的选项。它应当是原版人物或同道系统根据角色状态自动显示的交互入口。

当前结论：只要阿雅作为 GearMate/同道进入普通人物详情页，就可能暴露这类原版交互。因此阿雅随行不应使用 GearMate，而应保持为固定人物并通过专属事件交互。

## 当前首版边界

本版目标收敛为：

- 阿雅固定出现在太吾附近。
- 点击阿雅进入专属事件。
- 净化太吾当前地块内有玄灰或相枢感染的可治疗 NPC。
- 显示被净化者姓名。
- 成功后进入 6 个月冷却。
- 开放本 Mod 自己的随行/暂候，不开放原版同道、遣散、战斗参与。
- 自定义 PNG 立绘还未真正接入，只先整理资源与接口分析。
