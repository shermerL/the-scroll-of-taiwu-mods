# 太吾好伙伴阿雅 设计记录

## 角色方向

阿雅是“太吾好伙伴”系列的第一位特殊伙伴。她不是借用原版剧情人物，也不复用铜生的剧情字段；只借用官方“固定人物创建、入队、离队、事件对白、人物交互”的接口与流程。

角色气质参考“神兵小将”里南宫问雅一类的设定：天真、善良、容易相信别人，会把“找哥哥”和“帮大家恢复原样”挂在心上。为了避免直接搬运原作设定，Mod 中使用原创化名字“阿雅”，对白只保留可爱的语气方向。

## 官方事件包路线

首版应使用官方 `EventPackages`：

```text
Events/
  EventLib/Taiwu_EventPackage_GoodPartnerAya.dll
  EventLanguages/Taiwu_EventPackage_GoodPartnerAya_Language_CN.txt
  EventScript/ 可选
```

事件包负责：

- 初遇对白
- 是否带她同行的选项
- 点击阿雅后的专属交互选项
- 净化玄灰后的反馈对白

后端插件负责：

- 判断触发时机
- 创建并记录阿雅角色 id
- 调用官方入队/离队接口
- 存档内记录已触发、已入队、已治疗等状态
- 执行玄灰治疗逻辑

`.twes` 脚本首版可不做，优先把逻辑写在 C# 事件回调里，正文和选项文本走语言文件。

当前实现状态：

- 已用手写 C# 事件包接入 `Events/EventLib/Taiwu_EventPackage_GoodPartnerAya.dll`。
- `Config.Lua` 已声明 `EventPackages = {"Taiwu_EventPackage_GoodPartnerAya.dll"}`。
- 初遇头事件挂在 `TaiwuBlockChanged`，只在太吾村之后、非战斗、非奇遇、非月结、没有事件正在显示时检查。
- 固定人物点击事件挂在 `FixedCharacterClicked`，只响应当前记录的阿雅角色 id。
- 阿雅的人物模板由 `Config/Character_Aya.lua` 随 Mod 内置：复制普通“京畿女”配置，改为固定人物模板 `TaiwuGoodPartnerAya.Aya` / `12001`，不借用铜生等原版剧情人物字段。

## 初遇与入队

触发时机定为“太吾已到达太吾村、进入自由行动、且官方事件队列空闲之后”的第一个安全时机。后端插件只负责检查条件和排队触发，不抢占官方剧情。

安全条件：

- 存档未触发过阿雅初遇。
- 当前太吾角色、太吾位置、太吾村数据均有效。
- 不在战斗、奇遇、月结动画、传剑流程、教程引导或官方剧情事件处理中。
- 当前没有官方事件正在显示或等待处理。
- 如果条件不满足，本次跳过，等下一次自由行动 tick、月初/日切或地图行动结束后再检查。
- 一旦成功排队初遇事件，立即写入 `TaiwuGoodPartnerAya.IntroQueued` 或 `TaiwuGoodPartnerAya.IntroTriggered`，避免重复触发。

剧情意图：

- 阿雅正在寻找哥哥。
- 她听说太吾会去很多地方，便请求同行。
- 太吾可以选择带上她，也可以暂不带。

入队设计：

- 阿雅必须创建为固定模板角色。
- 同意后调用官方 `JoinGroup`，让她作为特殊同道加入。
- 官方 `JoinGroup` 对固定模板角色有专门分支：设置 `LeaderId` 后调用 `DomainManager.Character.AddSpecialGroupCharacter(context, taiwuCharId, charId)`，并在加入普通 `_groupCharIds` 前返回。
- 因此阿雅按固定模板角色实现时，不占用普通同道名额。
- 如果固定模板或特殊同道流程失败，应终止入队并提示错误，不退化成普通同道。
- 阿雅不应开放完整普通 NPC 菜单；优先通过专属官方交互选项承载功能。
- 阿雅应按“原版特殊固定同道”设计，而不是普通随机 NPC 同道。

## 特殊同道显示与遣散

原版里部分特殊同道并不只是普通队友：

- 在地图上有圆形人物图标或固定人物入口。
- 点击人物时走特殊侧边栏/人物交互，而不是完整普通 NPC 菜单。
- 遣散或离队后会回到太吾村、太吾当前位置，或剧情指定地点附近。
- 位置变化会触发 `FixedCharacterLocationChanged`，从而刷新地图表现。

阿雅应参考这类原版逻辑：

- 使用固定人物或转换后的固定人物身份保存，而不是临时随机 NPC。
- 入队时加入太吾特殊同道集合。
- 跟随太吾时位置同步到太吾当前位置。
- 遣散时调用官方离队流程后，把阿雅放回太吾村或太吾当前位置附近，并触发位置变化通知。
- 地图显示应保持原版圆形头像/固定人物入口，不另做自绘 UI。
- 点击阿雅时只显示阿雅专属交互，例如净化玄灰、同行/离队相关对白。
- 传剑时使用官方特殊同道转移逻辑；太吾传承到新人物后，阿雅应跟随新的太吾身份，而不是绑定旧太吾肉身。
- 如果阿雅当时正在同行，传剑后由官方 `TransferSpecialGroup` 把她迁到新太吾名下，并在下一次安全 tick 校验位置同步。
- 如果阿雅当时已被遣散，保持她在太吾村或上次记录地点，下一次交互时再按新太吾身份处理同行。
- 净化冷却属于阿雅自己的疲惫状态，传剑后不重置。

实现参考：

- `DomainManager.Taiwu.JoinGroup(context, charId)`
- `DomainManager.Taiwu.LeaveGroup(context, charId)`
- `DomainManager.Character.AddSpecialGroupCharacter(context, taiwuCharId, charId)`
- `DomainManager.Character.GetSpecialGroup(taiwuCharId)`
- `DomainManager.Character.TransferSpecialGroup(context, oldTaiwuCharId, newTaiwuCharId, clearDestGroupFirst: true)`
- `Events.RaiseFixedCharacterLocationChanged(context, charId, oldLocation, newLocation)`
- 原版 `GearMateJoinGroup` / `GearMateLeaveGroup` / `GearMateFollowTaiwu` 行为

## 同道协助与战斗命令

结论：阿雅仍应保持“特殊同道”路线，不占普通同道名额；同时给她配置原版预设同道协助命令，让她在可用场景中拥有类似普通同道的辅助行为。

目前从原版接口看，特殊同道和战斗协助并不冲突：

- `CharacterDomain.AddGroupCharactersToCombatTeam` 会在普通同道加入战斗队伍后，继续调用 `AddSpecialGroupCharactersToCombatTeam`。
- `AddSpecialGroupCharactersToCombatTeam` 会遍历 `GetSpecialGroup(taiwuCharId)`，符合条件的特殊同道可进入战斗队伍。
- `ExtraDomain.GetCharOriginalTeammateCommands` 对非随机创建角色会优先读取 `CharacterItem.PresetTeammateCommands`。
- 因此固定模板阿雅可以通过 `PresetTeammateCommands` 获得稳定的同道协助命令，不需要把她塞进普通 `_groupCharIds`。

首版建议预设命令：

| 命令 id | 原版含义 | 首版策略 |
| ---: | --- | --- |
| 7 | `HealPoison` / 祛毒 | 启用，符合阿雅“净化、照顾别人”的定位。 |
| 6 | `HealInjury` / 疗伤 | 启用，作为温和医护能力。 |
| 8 | `HealFlaw` / 破绽处理 | 暂不启用，后续平衡测试后再考虑。 |
| 9 | `HealAcupoint` / 点穴处理 | 暂不启用，后续平衡测试后再考虑。 |
| 13 | `TransferInjury` / 代伤 | 暂不启用，容易让角色定位过强。 |
| 15 | `AttackSkill` / 攻击技 | 暂不启用，阿雅首版不走战斗输出路线。 |

平衡与实现边界：

- 祛毒、疗伤是原版普通同道协助命令，不等同于阿雅的“净化玄灰”能力。
- 同道协助应继续走原版消耗、判定和禁用原因，例如草药资源、造诣、目标是否真的中毒或受伤。
- 阿雅的玄灰净化是点击阿雅后，通过官方人物交互和事件对白触发的独立剧情能力，不挂到战斗同道命令里。
- 玄灰净化仍使用 6 个月冷却；普通祛毒/疗伤不应绕过这个冷却，也不应治疗玄灰。
- 如果要保证祛毒按钮不是长期灰色，需要给阿雅配置合适但不夸张的医毒相关资质、造诣或资源。
- 首版不复制铜生的具体“丙阵”内容，只借用“特殊固定同道可以拥有预设协助命令”的做法。

需要实测的点：

- 当太吾已有 3 个普通同道时，阿雅作为特殊同道进入战斗队伍后，协助 UI 是否会显示她，还是受三人界面上限影响。
- 如果 UI 上限导致她挤占普通同道显示，应优先保留“不占普通同道名额”和“地图/事件净化”能力，战斗协助作为可关闭功能处理。
- 传剑、离队、重新入队后，`PresetTeammateCommands` 是否稳定保留。
- `HealPoison` 的资源和造诣门槛需要实测，避免首版看得到按钮但完全不可用。

## 首版能力：天生净化力量

这是首版确定实现的核心能力。

设定对应：

- 阿雅能把被污染、被魔化、被玄灰影响的人恢复清明。
- 在太吾规则里对应“治疗玄灰”或移除相关感染状态。

交互形式：

- 点击阿雅后出现专属交互，例如“请阿雅净化玄灰”。
- 对话进入原版事件界面。
- 选择确认后执行治疗。
- 该能力只在阿雅的人物交互/事件对白中体现，不作为战斗协助按钮，也不混入普通同道的祛毒、疗伤命令。
- 治疗后给一段可爱的反馈对白。
- 每次成功净化后，阿雅进入 6 个月冷却，文案表现为“累坏了，需要缓一缓”。
- 冷却期间净化选项应不可用，或进入疲惫对白，不消耗行动力和资源。

首版治疗范围：

- 优先采用“当前太吾所在格子可见 NPC 范围”。
- 范围包括当前地图格子的 `CharacterSet`，以及太吾当前队伍/特殊同道中实际跟随在同一位置的 NPC。
- 首版只治疗 NPC，不治疗太吾本人。
- 执行前扫描所有候选角色，找出确实存在玄灰或相枢感染状态的人。
- 如果候选中只有一个可治疗目标，就治疗这个人。
- 如果候选中有多个可治疗目标，就一次性一起治疗，并在对白里说明阿雅把大家都安抚下来了。
- 如果没有可治疗目标，选项应不可用，或进入“不用担心，大家现在都好好的”这类安慰对白，不消耗行动力或资源。

实现注意：

- 必须先确认目标确实存在玄灰或相枢感染状态。
- 判断条件至少包含 `GetXiangshuInfection() > 0` 或 `HasDarkAsh` / `GetDarkAshProtector()` 存在有效玄灰状态。
- 治疗相枢感染优先使用 `DomainManager.Character.RemoveXiangshuInfection(context, character, 0)`。
- 治疗玄灰优先使用 `character.RemoveDarkAsh(context)`。
- 成功治疗后可调用 `character.SavedFromInfected(context, batchMode: true)` 或同等原版记录方法，具体以测试效果为准。
- 不应治疗剧情锁定、死亡、不可交互或游戏明确禁止修改的角色。
- 不应治疗不可治疗角色，例如 `AllowHeal == false`、无效位置、临时敌人、尸体、剧情强锁定角色。
- 不应治疗太吾本人、阿雅本人、已死亡角色、事件临时敌人、不可交互角色，以及官方明确禁止修改的角色。
- 冷却建议记录为 `TaiwuGoodPartnerAya.PurifyCooldownUntilDate` 或后续新增的 `TaiwuGoodPartnerAya.LastPurifyDate`，以游戏内年月为准计算 6 个月，而不是现实时间。
- 冷却保存到存档级 Mod 数据，读档、换月、传剑后都保持有效。
- 批量治疗时单个角色失败不应中断整个流程；记录 warning 后继续处理其他目标。
- 只记录 warning/error，便于玩家反馈问题；正常治疗不刷日志。

## 后续待定：能与动物交流

这是很适合阿雅的后续扩展，但不放进首版承诺。

可能方向：

- 异兽互动：安抚被玄灰影响的异兽，降低敌意或触发特殊对白。
- 促织互动：识别促织状态、轻微改善心情或触发趣味事件。
- 坐骑互动：安抚坐骑，恢复状态或减少异常影响。
- 被玄灰影响的 NPC：通过“听见痛苦/害怕”的表达，引出净化剧情。

设计约束：

- 仍然走官方事件包和人物交互，不做额外大 UI。
- 不直接跳过核心玩法，不做万能收益按钮。
- 优先提供剧情味和小帮助，避免破坏平衡。

## 开局自带装备

原则：

- 只使用游戏现成物品模板，不新增魔改装备。
- 不给关键剧情物品、任务物品、特殊锁定物品。
- 不给过高品级装备，避免阿雅入队即破坏早期平衡。
- 装备要符合“善良、轻巧、净化、保护、寻找哥哥”的气质。

推荐方向：

- 衣着：选择低到中品级、颜色明亮或轻便的原版衣着。
- 饰品：选择朴素、护身、温和气质的原版饰品。
- 护具：选择轻甲或布甲方向，不走重甲压迫感。
- 武器：优先轻巧、非邪性、非毒系、非血腥的原版武器；如针、短杖、轻剑、符类等候选。
- 药品：可少量携带治疗或解毒类常规药物，用作“会照顾别人”的角色表达。

实现时需要从配置中筛选：

- `Weapon`
- `Armor`
- `Clothing`
- `Accessory`
- `Medicine`

筛选条件：

- `AllowRandomCreate` 或常规可创建来源优先。
- 排除 `IsSpecial`、剧情锁定、不可转移、任务相关物品。
- 品级建议控制在七品到五品之间，除非后续剧情解锁更高品级。
- 固定模板 id 写入文档后再进入代码，避免靠名字模糊匹配。

配置方式：

- 首版应附带一个本地安全配置，例如 `Config/AyaConfig.json`。
- 配置里记录装备模板 id、净化冷却月数、是否启用初遇事件等少量可维护项。
- 代码内保留一份安全默认值；配置缺失、格式错误、模板 id 不存在或命中特殊/剧情/任务物品时，忽略该项并回退到默认值。
- 不开放任意添加物品式配置，避免玩家误填关键剧情物品导致存档异常。
- 正式写代码前，需要从最新游戏配置中确认最终装备模板 id，并把候选与理由补到本文档。

首版默认配置文件：

- 路径：`Config/AyaConfig.json`
- 用途：作为打包随附的安全配置；代码内仍应保留同样的默认值作为兜底。

## 待确认问题

- 初遇事件排队 API 的最终调用点，需要实测确认具体钩子。
- 玄灰/相枢感染的全部边界状态需要用测试存档验证，避免误改剧情锁定角色。
- 默认装备 id 已从当前同步配置初筛；正式写代码时仍需在运行时校验模板存在、`IsSpecial == false`、`TaskLock` 为空。
